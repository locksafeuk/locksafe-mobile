ISSUE 1: PASSKIT / APPLE PAY FRAMEWORK
----------------------------------------

PassKit/Apple Pay is NOT actively used in this application.

WHY IT APPEARS:
• The app includes @stripe/stripe-react-native SDK for backend payment processing
• The Stripe SDK bundles Apple Pay capabilities by default
• merchantIdentifier "merchant.uk.locksafe.app" is configured as part of Stripe SDK initialization requirements

WHAT WE ACTUALLY USE:
• Server-side Stripe payment intents only (created on backend)
• Traditional card processing, NOT Apple Pay
• No Apple Pay buttons, payment sheets, or PKPayment APIs implemented anywhere in the app

CODE EVIDENCE:
• No usage of: initPaymentSheet(), presentApplePay(), confirmApplePayPayment()
• Payment service only creates backend payment intents via API calls
• Zero Apple Pay UI components in codebase

RESOLUTION:
We can remove the merchantIdentifier from config in Build 9 if required, or you may approve with this clarification. The entitlement exists due to SDK dependency, but no Apple Pay functionality is accessible to users.

ISSUE 2: BUSINESS MODEL QUESTIONS
-----------------------------------

OVERVIEW:
This is a B2B service provider app for professional locksmiths (similar to Uber Driver, DoorDash Dasher, TaskRabbit Tasker). NOT a consumer app.

DETAILED ANSWERS:

1) WHO ARE THE USERS?
   • Professional locksmiths (licensed tradespeople)
   • UK-based locksmith businesses
   • Verified through backend (license numbers, insurance)
   • Users do NOT purchase anything within this app

2) WHERE CAN USERS PURCHASE CONTENT/FEATURES?
   • Users (locksmiths) DO NOT purchase anything
   • NO in-app purchases, subscriptions, or payments
   
   HOW IT WORKS:
   - Customers visit www.locksafe.uk website
   - Customers pay assessment fee (£29-49) on WEB PLATFORM via Stripe
   - Job request appears in locksmith's app (already paid by customer)
   - Locksmith accepts job, performs physical service
   - Locksmith receives earnings via Stripe Connect bank transfer
   
   PAYMENT FLOW:
   Customer (Web) → Pays → Job Created → Locksmith (App) sees job → 
   Performs service → Receives payout to bank account
   
   Zero financial transactions occur within this mobile app.

3) WHAT PREVIOUSLY PURCHASED CONTENT CAN USERS ACCESS?
   • Not applicable - we don't provide "content"
   • Locksmiths access JOB REQUESTS (service opportunities), not content
   • They view: job details, customer location, earnings history
   • This is WORK OPPORTUNITIES, not unlockable content
   
   Nothing was "purchased" by locksmith to access jobs. Customers paid for 
   the SERVICE (locksmith call-out) on web platform.

4) WHAT PAID FEATURES ARE UNLOCKED WITHOUT IN-APP PURCHASE?
   • NONE - all features are FREE to locksmiths
   
   FREE FEATURES (No subscription/payment required):
   ✓ Job request notifications
   ✓ Job listings in coverage area
   ✓ Job acceptance and management
   ✓ GPS navigation
   ✓ Quote builder tool
   ✓ Photo documentation
   ✓ Digital signature capture
   ✓ Earnings tracker
   ✓ Job history
   
   NO PAID TIERS:
   • No Premium vs Basic accounts
   • No subscription plans
   • No feature unlocks
   • No content libraries

REVENUE MODEL (Commission-based Marketplace):
• Customer pays £40 assessment fee on WEBSITE
• Locksmith completes job using APP
• Locksmith receives £34 (85%) via Stripe Connect payout
• LockSafe retains £6 (15%) commission

Locksmith NEVER pays LockSafe through this app. Commission is deducted 
from customer payment before transfer. The app is completely free to use.
